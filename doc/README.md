# Документация проекta velo_weather

Велосипедный оптимизатор маршрута: прогноз времени и осадков по 10км-отрезкам с учётом
рельефа, направления, ветра и мощности педалирования. Flask-сервер + симулятор.

## Карта документации

| Документ | О чём |
|----------|-------|
| [overview.md](overview.md) | Цель, режимы работы, файлы проекта, быстрый старт, `config.json` |
| [architecture.md](architecture.md) | Физика `solve_speed`, направление ветра, планировщик остановок, дождь/без, API сервера |
| [weather.md](weather.md) | Open-Meteo, сетка точек, умное расписание фетча, дешёвая проба, `weather_log`/`weather_now` |
| [archive.md](archive.md) | Архив статистики заезда (`archive/<ride_id>/`) + стоп-гап текущей погоды + как анализировать |
| [deploy.md](deploy.md) | VPS, systemd-сервисы, команды деплоя |
| [decisions.md](decisions.md) | Журнал ключевых решений (почему так) + разборы логов |
| [todo.md](todo.md) | Задачи: активные и сделанные |

Корневой `claude.md` — краткий индекс для Claude Code, ссылается сюда.

## Как вести документацию

- **Дописывать по ходу дела.** Новая фича/решение → правим профильный файл, а не плодим новые.
- Факты «как работает» → `architecture.md` / `weather.md` / `archive.md`.
- «Почему так» и разборы данных → `decisions.md` (это журнал, добавляем сверху по дате).
- Задачи → `todo.md` (формат `#N`, активные сверху, сделанные с `~~зачёркиванием~~`).
- Операционные данные (логи, архив, `state.json`) в репозиторий **не коммитим** (см. `.gitignore`).
