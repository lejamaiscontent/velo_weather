# Проект: велосипедный оптимизатор маршрута

## Цель
Предсказать время и осадки по каждому 10км-отрезку 600км-маршрута, учитывая:
- фиксированное время старта
- мощность педалирования (Вт)
- рельеф и направление движения из GPX
- прогноз погоды (ветер, осадки) из Open-Meteo

Два режима работы:
1. **CLI** — вывод таблицы и графика (WIP)
2. **Веб-сервер** (`app.py`) — живой дашборд с пересчётом каждые 30 мин

## Архитектура core.py

### Физика (`solve_speed`)
```
P = [0.5·ρ·CdA·(v + v_wind)² + Crr·m·g·cos θ + m·g·sin θ] · v
```
Решается методом Ньютона. v_wind > 0 → встречный.

### Планировщик остановок (`simulate`)
На каждом сегменте:
1. **Ночёвка** — когда `cum_km ≥ overnight_km` и бюджет позволяет
2. **Дождевая стоянка** — если `precip ≥ rain_threshold`, смотрим прогноз вперёд на `max_rain_wait_h`: если дождь кончится — ждём, потом едем
3. Не превышаем `time_limit_h`

### Погода
API: **Open-Meteo** (без ключа, бесплатно, ≤3 дня)
Переменные: `wind_speed_10m`, `wind_direction_10m`, `precipitation`

## API сервера

| Endpoint | Метод | Описание |
|----------|-------|----------|
| `/` | GET | Главная страница с таблицей |
| `/update_km` | POST | Обновить текущий км (`form: km=123`) |
| `/recalc` | POST | Форсировать пересчёт |
| `/api/state` | GET | JSON: grid, last_calc, error |
| `/api/ride` | GET | JSON: полная симуляция по км |

## Фоновый цикл и сбор данных

- **Умное расписание фетча** (`_should_fetch_model`): тихая зона → медленный/быстрый поллинг. Параметры и логика — в `KNOWLEDGE.md`.
- **Дешёвая проба** (`_probe_model`): перед полным фетчем проверяет обновление модели по `A = min(20, ⌈√N⌉)` случайным точкам сетки; полный фетч (всех точек) — только при обнаружении изменения. Экономит ~6–10× запросов.
- **Две модели** (`icon_seamless`, `ecmwf_ifs025`) считаются параллельно; в UI — вкладки.
- **Логи:** `weather_log.jsonl` (каждая проверка: `new_run/mode/check/points`), `weather_now.jsonl` (текущая погода по маршруту раз в 30 мин, из кеша — без API), `power_log.jsonl` (мощность + входы калибровки).
- **Архив заезда** `archive/<ride_id>/`: `forecast.jsonl` (снимок прогноза на каждый пересчёт, с триггером), `position.jsonl` (факт. фиксы + прогноз, под которым ехали + lead), `meta.json` (config/GPX-sha1/git — реплей). Цель — офлайн-отладка точности скорости/ETA/мощности. Формат — в `KNOWLEDGE.md`.

## Известные ограничения
- Ветер на высоте 10м, рельефного укрытия нет
- Мощность постоянна (без учёта усталости)
- Осадки: линейная аппроксимация мм·ч
- Всё в UTC — старт тоже указывать в UTC
- GPX без высот → рельеф не считается (нет предупреждения — TODO)
