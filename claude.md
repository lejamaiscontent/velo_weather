# Проект: велосипедный оптимизатор маршрута

## Цель
Предсказать время и осадки по каждому 10км-отрезку 600км-маршрута, учитывая:
- фиксированное время старта
- мощность педалирования (Вт)
- рельеф и направление движения из GPX
- прогноз погоды (ветер, осадки) из Open-Meteo

Два режима работы:
1. **CLI** — вывод таблицы и графика (WIP)
2. **Веб-сервер** (`app.py`) — живой дашборд с пересчётом каждые 30 мин

## Файлы

| Файл | Описание |
|------|----------|
| `core.py` | GPX-парсинг, физика, погода, симуляция с планировщиком остановок |
| `app.py` | Flask-сервер; фоновый поток пересчёта; API `/api/state`, `/api/ride` |
| `templates/index.html` | Веб-морда: 10км-таблица, форма обновления текущего км |
| `config.json` | Параметры маршрута (GPX, старт, мощность и т.д.) |
| `state.json` | Кешированное состояние (генерируется автоматически) |
| `requirements.txt` | `gpxpy`, `requests`, `flask` |

## Быстрый старт

```bash
pip install -r requirements.txt
# положить route.gpx рядом с app.py
# заполнить config.json
python app.py
# открыть http://localhost:5000
```

## config.json

| Ключ | По умолчанию | Описание |
|------|-------------|----------|
| `gpx_file` | `route.gpx` | Путь к GPX |
| `start_time` | ближайший час UTC | ISO-строка `2025-07-01T06:00:00+00:00` |
| `power_w` | 150 | Мощность, Вт |
| `mass_kg` | 85 | Масса тело+вел+груз, кг |
| `cda` | 0.36 | Аэродинамика CdA, м² |
| `crr` | 0.004 | Коэффициент качения |
| `time_limit_h` | 40 | Лимит времени, ч |
| `overnight_km` | 300 | Км ночёвки |
| `overnight_h` | 8 | Длительность ночёвки, ч |
| `rain_threshold` | 0.5 | Осадки для остановки, мм/ч |
| `max_rain_wait_h` | 3.0 | Макс. ожидание дождя, ч |
| `weather_samples` | 10 | Точек для запросов погоды |
| `weather_model` | `icon_seamless` | `icon_seamless` или `ecmwf_ifs025` |
| `recalc_interval_min` | 30 | Интервал фонового пересчёта |

## Архитектура core.py

### Физика (`solve_speed`)
```
P = [0.5·ρ·CdA·(v + v_wind)² + Crr·m·g·cos θ + m·g·sin θ] · v
```
Решается методом Ньютона. v_wind > 0 → встречный.

### Планировщик остановок (`simulate`)
На каждом сегменте:
1. **Ночёвка** — когда `cum_km ≥ overnight_km` и бюджет позволяет
2. **Дождевая стоянка** — если `precip ≥ rain_threshold`, смотрим прогноз вперёд на `max_rain_wait_h`: если дождь кончится — ждём, потом едем
3. Не превышаем `time_limit_h`

### Погода
API: **Open-Meteo** (без ключа, бесплатно, ≤3 дня)
Переменные: `wind_speed_10m`, `wind_direction_10m`, `precipitation`

## API сервера

| Endpoint | Метод | Описание |
|----------|-------|----------|
| `/` | GET | Главная страница с таблицей |
| `/update_km` | POST | Обновить текущий км (`form: km=123`) |
| `/recalc` | POST | Форсировать пересчёт |
| `/api/state` | GET | JSON: grid, last_calc, error |
| `/api/ride` | GET | JSON: полная симуляция по км |

## Известные ограничения
- Ветер на высоте 10м, рельефного укрытия нет
- Мощность постоянна (без учёта усталости)
- Осадки: линейная аппроксимация мм·ч
- Всё в UTC — старт тоже указывать в UTC
- GPX без высот → рельеф не считается (нет предупреждения — TODO)
